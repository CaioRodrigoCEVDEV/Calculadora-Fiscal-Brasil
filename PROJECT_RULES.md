# PROJECT_RULES.md

Regras gerais do projeto Calculadora Fiscal Brasil.

## Objetivo do projeto

- Disponibilizar calculadoras fiscais gratuitas, simples e confiáveis para usuários brasileiros.
- Ajudar usuários a entender tributos, simulações e memórias de cálculo.
- Gerar tráfego orgânico por SEO e monetização via AdSense.

## Regras gerais

- Não transformar a aplicação em sistema complexo sem necessidade.
- Priorizar velocidade, simplicidade e SEO.
- Evitar dependências desnecessárias.
- Manter código legível.
- Evitar overengineering.
- Preservar o foco em páginas públicas de alto valor utilitário.

## Regra para novas calculadoras

Toda nova calculadora deve ter:

- Formulário claro.
- Resultado objetivo.
- Memória de cálculo.
- Aviso fiscal/legal.
- Conteúdo explicativo para SEO.
- Estrutura que permita entendimento por leigos e conferência por profissionais.

## Público-alvo

- Empreendedores.
- Contadores.
- Estudantes.
- Analistas fiscais.
- Usuários pesquisando cálculos de impostos no Google.

## Regras de conteúdo

- Usar linguagem simples.
- Evitar prometer exatidão legal absoluta.
- Sempre recomendar conferência com contador e legislação vigente.
- Não usar jargão excessivo sem explicação.
- Priorizar utilidade prática e clareza textual.

## Regras de produto

- Este projeto não deve virar ERP, sistema de gestão ou plataforma fiscal complexa sem demanda explícita.
- O valor principal está em simulação rápida, explicação clara e confiança.
- Sempre que possível, mostrar a lógica do cálculo em vez de apenas o número final.

## Versionamento

- Versão pública atual: `1.0.0`.
- Alterações relevantes devem sugerir nova tag Git.
- Usar versionamento semântico:
- `patch`: correções pequenas.
- `minor`: novas features ou novas calculadoras.
- `major`: mudanças estruturais ou incompatíveis.
